# Аудит левой панели настроек ClickForge

## 1. Структура и состав элементов

### SidebarView (верх → низ)

| № | Элемент | Описание |
|---|---------|----------|
| 1 | Заголовок | waveform + "ClickForge" |
| 2 | Divider | Разделитель |
| 3 | SettingsPanelView | Вся панель настроек (ScrollView) |
| 4 | Spacer | Растягивает контент к верху |
| 5 | Футер | Статус движка (зелёный/оранжевый) + CPU/MB |

### SettingsPanelView (внутри ScrollView)

| № | Элемент | Тип | Условие показа |

## 2. Размещение и порядок

### Плюсы текущего порядка

1. **BPM/Тональность вверху** — ключевой контекст для сессии, логично показывать первым при наличии анализа.
2. **Клик → Пре-клик → Голос** — логичная цепочка настроек звука клика (что добавляем к трекам).
3. **Формат вывода** — перед опциями обработки, т.е. «что экспортируем» до «как экспортируем».
4. **Обработка** — «Создать метроном-трек» и «Анализ BPM по всем трекам» рядом, оба относятся к обработке.
5. **Устройство вывода** — отдельно, это про воспроизведение (playback), не про экспорт.
6. **Выходная папка** — внизу, относится к экспорту, после выбора формата и опций.

### Рекомендации по порядку

| Проблема | Предложение |
|----------|-------------|
| «Устройство вывода» между «Обработка» и «Выходная папка» | Можно перенести «Устройство вывода» **после** «Выходная папка» — тогда: сначала экспорт (формат, опции, папка), потом воспроизведение (устройство). Или оставить как есть — устройство влияет и на воспроизведение, и на предпросмотр. |
| «Выходная папка» условно внизу | Корректно: папка задаётся папкой проекта, показывать её имеет смысл только после выбора. |

**Вывод**: Порядок в целом логичный, серьёзных изменений не требуется.

## 3. Визуальное представление

### 3.1 Отступы (неконсистентность)

| Область | Горизонтальный padding | Вертикальный |
|---------|------------------------|--------------|
| Заголовок Sidebar | 16 | 12 |
| SettingsPanelView (ScrollView) | 12 | top: 8 |
| Футер Sidebar | 16 | bottom: 12 |

**Проблема**: Внутри ScrollView используется `padding(.horizontal, 12)`, а в шапке и футере — `16`. Визуально панель настроек уже заголовка и футера.

**Рекомендация**: Унифицировать — либо везде 16, либо везде 12. Предпочтительно **16** для единообразия с заголовком и футером.

### 3.2 Секционные заголовки

| Секция | Шрифт | Цвет | Стиль |
|--------|-------|------|-------|
| Настройки | .caption.bold | .secondary | Один общий заголовок |
| Формат вывода | .caption2 | .secondary | ✓ |
| Обработка | .caption2 | .secondary | ✓ |
| Устройство вывода | .caption2 | .secondary | ✓ |
| Выходная папка | .caption2 | .secondary | ✓ |

**Проблема**: «Настройки» — `.caption.bold`, подзаголовки секций — `.caption2` без bold. Иерархия есть, но «Настройки» почти дублирует назначение всей панели и может быть избыточным.

**Рекомендация**: Либо убрать «Настройки» (контекст и так понятен), либо оставить как основной заголовок — тогда лучше сделать его `.headline` или `.title3` для чёткого отличия от подзаголовков.

### 3.3 Блок BPM + Тональность

- Фон: `Color.accentColor.opacity(0.08)` ✓ (соответствует UI_AUDIT)
- Padding: 10 ✓
- CornerRadius: 8 ✓
- Две колонки: BPM слева, Тональность справа ✓

**Плюс**: Хорошо выделяет ключевую информацию. Минус: при отсутствии анализа блок исчезает — заголовок «Настройки» сразу переходит к «Клик дБ». Это нормально, контент адаптивен.

### 3.4 Элементы управления

| Элемент | Макет | Проблема |
|---------|-------|----------|
| Клик дБ | LabeledContent + Slider + текст "X dB" (width: 44) | ✓ Согласовано |
| Голос дБ | То же | ✓ |
| Пре-клик | LabeledContent + Stepper | ✓ |
| Формат | Picker radioGroup, labelsHidden | ✓ |
| Устройство | Picker menu, labelsHidden | ✓ |

**Замечание**: Слайдеры «Клик дБ» и «Голос дБ» оформлены одинаково — консистентно ✓

### 3.5 Dividers

Используется **5 разделителей** внутри SettingsPanelView. Они разделяют:

1. BPM блок (если есть) → Клик
2. Голос → Формат вывода
3. Формат → Обработка
4. Обработка → Устройство
5. Устройство → Выходная папка (если есть)

**Вывод**: Разделение секций логичное, визуальная нагрузка приемлемая.

## 4. Рациональность структуры

### Группировка по смыслу

| Группа | Элементы | Оценка |
|--------|----------|--------|
| Контекст сессии | BPM, Тональность | ✓ |
| Звук клика | Клик дБ, Пре-клик, Голос | ✓ |
| Экспорт | Формат, Обработка | ✓ |
| Воспроизведение | Устройство вывода | ✓ |
| Назначение вывода | Выходная папка | ✓ |

Структура соответствует логике приложения.

### Условная видимость

| Условие | Поведение |
|---------|-----------|
| `state.analysis != nil` | Показать BPM блок |
| `state.voiceEnabled` | Показать «Голос дБ» |
| `!state.folderPath.isEmpty` | Показать «Выходная папка» |
| `state.audioDevices.isEmpty` | «Нет доступных устройств» вместо Picker |

Условная логика обоснована и не создаёт лишних элементов.

### Футер (статус движка)

- Зелёный/оранжевый индикатор — понятная семантика ✓
- «Движок готов» / «Запуск…» — ясные состояния ✓
- CPU/MB — только когда движок готов ✓

Футер информативен и не перегружен.

## 5. Резюме и приоритеты

### Критические проблемы

**Нет.** Структура панели логична, элементы размещены рационально.

### Рекомендации (улучшения)

| Приоритет | Проблема | Решение |
|-----------|----------|---------|
| Средний | Несогласованность padding (12 vs 16) | Унифицировать horizontal padding до 16 для SettingsPanelView |
| Низкий | Заголовок «Настройки» | Убрать или выделить (например, .headline), чтобы не дублировать контекст |
| Низкий | Порядок «Устройство» / «Папка» | Рассмотреть обмен местами — папка перед устройством, т.к. папка относится к экспорту вместе с форматом и опциями |

### Состояние на текущий момент

- **Порядок элементов**: логичный ✓  
- **Визуальная иерархия**: читаемая ✓  
- **Консистентность**: в целом соблюдена, кроме padding ✓  
- **Рациональность**: хорошая ✓  

Левая панель в рабочем состоянии; изменения носят косметический характер и не обязательны для корректной работы.
