# Очевидные вещи, которые стоит доработать

## 1. Предупреждение перед обработкой при 0 активных треков
**Сейчас:** Если все треки muted (и нет solo), обработка всё равно запускается и возвращает «Экспортировано: 0 файлов» (кроме METRONOME при create_metro).
**Нужно:** Проверять до вызова process: есть ли хотя бы один активный трек (не muted) или группа с активными треками. Если нет — не запускать процесс и показать «Нет активных треков для экспорта».

## 2. Кнопка «Обновить папку» / пересканирование
**Сейчас:** Чтобы подхватить новые/удалённые файлы, нужно заново открыть папку.
**Нужно:** Кнопка обновления (например, рядом с выбором папки) для пересканирования текущей папки без повторного выбора.

## 3. Сброс errorMsg при новой операции
**Сейчас:** Предыдущая ошибка остаётся до закрытия alert. При новом process/play старый alert может вводить в заблуждение.
**Нужно:** Сбрасывать `errorMsg = nil` в начале process(), playAll(), scanFolder(), analyzeFile().

## 4. Отдельный alert при missing-файлах
**Сейчас:** Missing отображаются только в statusMsg после «Готово».
**Нужно:** Если `result.missing` не пустой — показывать предупреждающий alert: «Файлы не найдены: X, Y. Возможно, удалены или переименованы».

## 5. Валидация BPM
**Сейчас:** Проверяется только `bpm > 0`.
**Нужно:** Лимиты 20–300 BPM (или 10–400) — защита от некорректных значений и слишком быстрых/медленных темпов.

## 6. Горячие клавиши
**Сейчас:** Только Пробел — воспр./пауза.
**Можно добавить:**
- Cmd+O — открыть папку
- Cmd+R — обновить папку
- Cmd+Enter — обработать (если доступно)

## 7. Очистка папки назначения — только аудио?
**Сейчас:** Удаляются все файлы в папке.
**Вопрос:** Если пользователь положил туда что-то вручную (readme, другие файлы), это тоже удалится. Можно ограничить удаление только поддерживаемыми форматами (*.mp3, *.wav и т.д.) — по желанию.
